教程 1：模拟分布式算法

通过 Master/Workers 示例走通完整 SimGrid 模拟流程。两类 Actor：master 分发任务，worker 执行。默认轮询调度。

Master

static void master(std::vector<std::string> args) {
  long tasks_count = std::stol(args[1]);
  double compute_cost = std::stod(args[2]);
  long communication_cost = std::stol(args[3]);
  std::vector<sg4::Mailbox*> workers;
  for (unsigned int i = 4; i < args.size(); i++)
    workers.push_back(sg4::Mailbox::by_name(args[i]));
  for (int i = 0; i < tasks_count; i++)
    workers[i % workers.size()]->put(new double(compute_cost), communication_cost);
  for (auto* m : workers) m->put(new double(-1.0), 0);  // 终止信号
}

Worker

static void worker(std::vector<std::string> args) {
  sg4::Mailbox* mailbox = sg4::Mailbox::by_name(sg4::this_actor::get_host()->get_name());
  double cost;
  do { auto msg = mailbox->get_unique<double>(); cost = *msg;
       if (cost > 0) sg4::this_actor::execute(cost);
  } while (cost > 0);
}

main + 运行

int main(int argc, char* argv[]) {
  sg4::Engine e(&argc, argv);
  e.register_function("master", &master);
  e.register_function("worker", &worker);
  e.load_platform(argv[1]); e.load_deployment(argv[2]); e.run();
}

教程 2：形式化验证与模型检验

Mc SimGrid 是嵌入 SimGrid 的全功能模型检验器（model checker），可穷举探索分布式程序的所有可能执行路径，发现死锁、断言失败、竞态条件等 bug。

核心概念

  • 软件模型检验：不依赖数学建模，直接把程序本身当作模型来验证
  • 无状态模型检验：通过 UNIX fork 穷举所有执行分支
  • DPOR（动态偏序约简）：减少需探索的冗余路径
  • 关键转换（Critical Transition）：从此步之后所有执行都走向 bug
  • 时间在模型检验模式下变为离散的步骤（无性能时间）

两种观测方案

方案可观测事件适用
sthread（LD_PRELOAD 注入）pthread 操作、互斥锁、信号量等多线程 C/C++ 程序
SMPIMPI 通信操作MPI 程序
mclang（v4.1+，编译器插桩）内存读写事件，检测数据竞态多线程程序

运行方式

# 多线程程序（sthread 注入）
simgrid-mc --sthread ./my_program

# MPI 程序
smpirun -wrapper simgrid-mc -np 4 -platform p.xml ./my_mpi_program

# 重放出错的执行路径（获取详细回溯）
./my_program --cfg=model-check/replay:'1;2;1;1;2;4;1;1;3;1'

示例 1：哲学家就餐死锁

5 个哲学家、5 把叉子——如果所有哲学家同时拿起左手叉子，死锁。常规运行 10000 次也很难触发。用 simgrid-mc --sthread 运行，0.1 秒内就检测到死锁，并输出完整执行路径和关键转换点。

示例 2：数据竞态检测

# 用 mclang 编译（自动插桩内存访问）
mclang plusplus.c -o plusplus -g
# 输出: Successfully instrumented 4 read operations and 3 writes operations

# 模型检验
simgrid-mc --sthread ./plusplus
# 检测到: datarace at location 0x... between actor 2 and 3
# First operation was a WRITE, Second was a READ — both at plusplus.c:8

示例 3：非确定性消息接收（S4U/MPI）

Server 假设收到的最后一条消息来自 client 3。模型检验器发现消息顺序是不确定的——取决于链路速度，任何顺序都可能。通过穷举探索找出导致断言失败的执行路径。

局限性

  • 只观测已插桩的事件（MPI/线程/内存），不观测磁盘 I/O 和普通通信
  • 状态空间随 Actor 数和事件数指数增长——需精简验证程序
  • 时间为离散步骤，无法评估性能

SMPI：模拟 MPI 应用

运行未修改的 MPI 程序（C/C++/Fortran）。用 smpicc 替代 mpicc，smpirun 替代 mpirun——主要区别是多一个平台文件。

编译运行

smpicc prog.c -o prog       # C
smpicxx prog.cpp -o prog     # C++
smpirun -hostfile h.txt -platform p.xml ./prog
cmake -DMPI_C_COMPILER=/opt/simgrid/bin/smpicc .

默认 dlopen 方式（多次加载可执行文件，快）。可选 mmap（慢但更通用）。用 --cfg=smpi/privatization:no|dlopen|mmap 控制。

简介

SimGrid 是一个分布式应用在分布式平台上执行的模拟器开发框架。

四大用途

  1. 方案对比：不需真实硬件
  2. 平台协同设计：修改平台文件远比搭真实平台容易
  3. 调试真实应用：可复现的"全知视角"
  4. 形式化验证：穷举执行路径找 bug

项目组成

组件说明
应用C++/Python/C API 或 MPI 程序
模拟平台XML/C++ 硬件描述，支持动态行为
部署Actor → Host 映射
平台模型核心：精确建模并发通信的网络竞争

局限性

  • 专注分布式系统，不适用实时多线程
  • 不支持 5G/LoRa
  • 抽象掉 L2/L3 缓存、内核调度器

成功案例

  • 3,000+ 论文引用
  • Tibidabo ARM 集群建前就用 SimGrid 预测加速比
  • 调试 BigDFT、StarPU、TomP2P

S4U 接口指南

S4U 是 SimGrid 的现代 C++ 接口，从 v3.33 起是主接口。Actor → Activity（Exec/Comm/Io）→ Resource（Host/Link/Disk）。

概念类说明
执行流Actor模拟中独立执行单元
引擎Engine全局单例
通信汇聚Mailbox名字匹配 send/recv，不涉及网络延迟

Mailbox 通信

Mailbox::put(data, size);                       // 阻塞
CommPtr c = Mailbox::put_async(data, size);     // 异步
c->wait(); c->test(); c->wait_for(5.0);
Comm::sendto(src, dst, data, size);             // 直接通信

网络模型对比

模型特点适用
LV08TCP窗口+慢启动(×13.01)+ACK竞争+bf=0.97+wS=20537通用TCP
CM02仅TCP窗口，可解释性强简单TCP

Constant固定1秒抽象算法
Raw简单公平共享PCIe总线

LV08 详解

latency-factor=13.01（慢启动）、bandwidth-factor=0.97（TCP头开销）、weight-S=20537（RTT不公平性）、cross-traffic=0.05（ACK反向占用5%带宽）。A→B 100kB/1Mbps/10ms: 简单模型 0.81s → LV08 0.996s

校准

真实平台基准测试 → 测量不同消息大小延迟 → 计算校正因子 → 验证误差。
