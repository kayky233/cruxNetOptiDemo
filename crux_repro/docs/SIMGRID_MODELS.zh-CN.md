# SimGrid 性能模型

> 原文: https://simgrid.org/doc/latest/Models.html

SimGrid 的性能模型计算所有 Activity 的持续时间，基于平台特性和并发资源使用情况。

---

## 模拟的资源类型

| 资源 | 模型 | 说明 |
|---|---|---|
| 网络 | LV08（默认）、CM02、SMPI、IB、ns-3 | 持续改进近 20 年，经过反复验证 |
| CPU | Cas01 | 时间 = 计算量 / 速度，考虑核心数，公平时间共享 |
| 磁盘 | S19 | 较新，支持校准 |

---

## LMM 求解器（大多数模型的基础）

大多数 SimGrid 模型基于**线性最大最小（LMM）求解器**：

1. 每个 Activity 在模拟内核中表示为一个 **Action**，包含初始工作量和剩余工作量
2. 在每个模拟时间步，按 Max-Min 目标函数计算每个 Action 的瞬时速率
3. 约束条件：同一资源上所有 Action 的瞬时消耗之和 ≤ 资源标称能力
4. 计算最早完成的 Action，推进模拟时间，更新剩余工作量
5. 完成的 Action 释放对应 Actor

## 网络模型对比

| 模型 | 特点 | 适用场景 |
|---|---|---|
| **LV08**（默认）| 建模 TCP 窗口、慢启动、ACK 竞争。latency-factor=13.01, bandwidth-factor=0.97, weight-S=20537 | 通用 TCP 网络 |
| **CM02** | 仅 TCP 窗口，无校正因子。结果偏保守但可解释性强 | 需要可理解的 TCP 预测 |
| **SMPI** | 针对 HPC 优化，按消息大小分段校正，建模 eager/rendezvous 模式 | MPI 应用模拟 |
| **IB** | InfiniBand 网络，建模 IB 特有的竞争行为 | HPC + InfiniBand |
| **Constant** | 所有通信固定 1 秒 | 抽象算法研究 |
| **Raw** | 最简单的带宽公平共享，无 TCP 效应 | 机内总线（如 PCIe）|
| **ns-3** | 包级网络模拟器，精度最高速度最慢 | 需要高精度网络模拟 |

选择网络模型：

```bash
--cfg=network/model:Lv08     # 默认
--cfg=network/model:SMPI     # MPI 场景
--cfg=network/model:CM02     # 简单 TCP
```

## TCP 窗口机制

SimGrid 建模 TCP 窗口如下：

```
real_BW = min(physical_BW, TCP_GAMMA / (2 × latency))
```

- `TCP_GAMMA` 默认 4194304（4 MiB），可通过 `network/TCP-gamma` 配置
- 延迟足够大时，TCP 窗口限制了有效带宽
- 设置为 0 可禁用窗口机制

**示例**: 10Gb/s 链路
- 延迟 = 0: 物理带宽不受限，1 秒完成
- 延迟 = 0.00001s: 窗口限制带宽 ~209Gb/s > 10Gb/s，物理带宽仍为瓶颈
- 延迟 = 0.001s: 窗口限制带宽 ~2Gb/s < 10Gb/s，通信耗时约 4.77s
- 延迟 = 0.1s: 窗口限制带宽 ~21Mb/s，通信耗时约 477s

## LV08 模型详解（默认）

考虑主机 A 向 B 发送 100kB，链路带宽 1Mbps、延迟 10ms：

**简单模型**: 0.01 + 8e5/1e6 = **0.81 秒**

**LV08 模型考虑三个效应**:

1. **TCP/IP 头部开销** — 带宽因子 0.97，即实际可用带宽为标称的 97%
2. **慢启动** — 延迟因子 13.01，即有效延迟 = 物理延迟 × 13.01
3. **ACK 竞争（Cross-traffic）** — 反向 ACK 流额外消耗 5% 带宽

**LV08 计算结果**: 0.01×13.01 + 800000/((0.97×1e6)/1.05) = **0.996 秒**

## 带宽和延迟校正因子

三个关键配置项，可以按消息大小分段设置：

### network/latency-factor

默认 LV08: **13.01**（常数）

SMPI 默认值（分段）:
```
65472:11.6436;15424:3.48845;9376:2.59299;5776:2.18796;
3484:1.88101;1426:1.61075;732:1.9503;257:1.95341;0:2.01467
```

### network/bandwidth-factor

默认 LV08: **0.97**（常数）

SMPI 默认值:
```
65472:0.940694;15424:0.697866;9376:0.58729;5776:1.08739;
3484:0.77493;1426:0.608902;732:0.341987;257:0.338112;0:0.812084
```

### network/weight-S

RTT 不公平性权重：大 RTT 的连接在瓶颈链路上被惩罚。
- CM02: 0（公平共享）
- LV08: 20537
- SMPI/IB: 8775

## 校准模型

> 详见原文: [Calibrating the models](https://simgrid.org/doc/latest/Models_calibration.html)（页面暂不可访问）

要获得高精度模拟结果，**必须对你的具体环境校准模型**。校准步骤：

1. 在真实平台上运行基准测试
2. 测量不同消息大小下的通信延迟
3. 计算校正因子（latency-factor、bandwidth-factor、weight-S）
4. 用校准后的参数运行模拟，验证误差在可接受范围

## 选择模型指南

```
你的场景是什么？
├── 研究抽象分布式算法 → Constant 或 Raw
├── 模拟普通 TCP 网络应用 → LV08（默认）
├── 模拟 MPI 应用 → SMPI
├── 模拟 InfiniBand HPC → IB
├── 模拟机内 PCIe 等总线 → Raw
└── 需要极高网络精度 → ns-3（需额外安装）
```

## 本团队使用情况

当前 Python 模拟器 (`crux_sim.py`) 使用简化的权重竞争模型（非 SimGrid 原生模型）。真实 SimGrid C++ 实验 (`collective_sim.cpp`) 使用 LV08 默认模型。后续改进计划包括用 HCCL benchmark 校准 SMPI 或 LV08 模型的校正因子，使模拟时间与实际昇腾集群测量值一致。
